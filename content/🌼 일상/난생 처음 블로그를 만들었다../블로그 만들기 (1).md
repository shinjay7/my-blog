## 1. 준비 단계

우선 옵시디언을 가지고 블로그를 만들기 전에 뭘 준비해야 하는지부터 정리했다. 사실 과정이 일반 블로그들 보다 좀 많이 복잡하긴 하지만 새로운걸 시도해본다는 생각에 신나게 달려 들었다.  

준비물은 다음과 같다.  

- **[Obsidian](http://obsidian.md/)** : 블로그의 글이 될 `.md` 파일을 쓰고 수정하는 에디터.
- **[GitHub](https://github.com/)** : 블로그 저장소를 올려둘 공간(?). 마치 창고 같은 느낌.  
- **[Quartz](https://quartz.jzhao.xyz/)** : Obsidian 노트를 예쁘게 블로그로 변환시켜주는 도구.  
- **[Node.js + npm](https://nodejs.org/ko/)** : Quartz를 돌리려면 필요한 기반 도구. (사실 나도 잘 모른다..) 
- **[Git](https://git-scm.com/downloads)** : 버전 관리의 기본이자 Github과 대화하는(?) 도구. Quartz 설치시 리포를 `clone` 할때도 필요.
- **[Cloudflare Pages](https://www.cloudflare.com/ko-kr/)** : 내가 만든 블로그를 세상 밖으로 뿌려주는 배포 서비스. GitHub와 연동만 해두면, push 할 때마다 자동으로 웹사이트를 업데이트해준다.  (GitHub Pages도 있지만, Cloudflare가 좀 더 빠르고 유연하다. 게다가 무료 플랜도 넉넉하고, 개인 도메인 붙이기도 쉽다.)  

그리고 나는 맥북에어와 [해킨토시](https://namu.wiki/w/%ED%95%B4%ED%82%A8%ED%86%A0%EC%8B%9C), 이렇게 두 대를 같이 쓰고 있다. 그래서 양쪽 모두에서 원할한 작업을 하기 위해 블로그의 글이 연동될 수 있도록 iCloud Drive에 옵시디언 Vault를 동기화 했다.  

내 워크플로우를 정리하자면 다음과 같다.  

1. Obsidian으로 글을 작성 (iCloud 연동)  
2. Github에 Push  
3. 연동된 Quartz가 자동으로 빌드  
4. 연동된 Cloudflare Pages가 자동으로 배포  

## 2. 설치 과정 및 세팅

### 1. Homebrew 설치 (맥 환경 필수)
맥에서 어떤 프로그램을 설치한다? [Homebrew](https://brew.sh)가 가장 편하고 관리도 쉬운거 같다.  

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 2. Git 설치
버전 관리 및 Github과 대화하는(?) 도구.  

```bash
brew install git
git --version
```  

### 3. Node.js & npm
Quartz를 돌릴때 필요한 도구. Homebrew를 통해 원하는 특정 버전의 node를 다운 받을수도 있다. 나는 그냥 `brew install node` 로 설치했다.  

```bash
brew install node
node -v
npm -v
```  

### 4. Obsidian 설치
이번 블로그 만들기의 핵심!  

```bash
brew install --cask obsidian
```  

### 5. Quartz 설치
옵시디언 노트를 블로그 형태로 전환해주는 중요한 툴!  

```bash
git clone https://github.com/jackyzha0/quartz.git
cd quartz
npm i
npx quartz create
```  

Quartz 설치가 완료되면 `quartz` 폴더 안에 `content` 폴더가 존재하는데, 옵시디언 vault의 심볼릭 링크를 생성하여  `content` 폴더를 대체하면 된다.

```bash
ln -s '옵시디언 vault path' content
```  

이때 기존 `content` 폴더 안에는 `index.md` 파일이 존재하는데, 이 파일은 새로운 `content` 폴더 안에도 **무조건** 존재해야한다. 그래야 `quartz` 가 정상적으로 가동된다.  

### 6. Github과 Cloudflare 연결하기
이 [블로그 글](https://anpigon.pages.dev/%F0%9F%A7%B0-%EC%83%9D%EC%82%B0%EC%84%B1-%EB%8F%84%EA%B5%AC/%EC%98%B5%EC%8B%9C%EB%94%94%EC%96%B8-Obsidian/Quartz/%EC%98%B5%EC%8B%9C%EB%94%94%EC%96%B8-%EB%AC%B4%EB%A3%8C-%ED%8D%BC%EB%B8%94%EB%A6%AC%EC%8B%9C-%EB%B0%A9%EB%B2%95(Quartz))을 참고하여 진행하였다. 설명이 매우 쉽게 잘되어 있다.  

### 7. 배포
터미널에서 `quartz` 폴더로 이동한 후 `npx quartz sync` 를 하면 배포가 자동으로 진행된다.  
```bash
cd 'quartz path'
npx quartz sync
```
